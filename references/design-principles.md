## 排版优先哲学

> 研究来源：[beautiful-html-templates](https://github.com/zarazhangrui/beautiful-html-templates)（2637+ stars）——专业级 HTML 演示模板的 12 个核心设计模式。

**核心原则：字体选择就是人格，颜色就是设计，装饰是最后的手段。**

专业级演示文稿的视觉冲击力不来自装饰元素——它来自：
1. **大胆的字体选择**——每个模板用独特的字体组合定义人格
2. **自信的色彩决定**——整页纯色背景，不是装饰性渐变
3. **极度的克制**——1 种强调色，零装饰图形

### Token 驱动的设计系统

所有视觉属性通过 CSS 自定义属性定义，绝不硬编码。每个模板有完整的 token 系统：

```css
:root {
  /* 颜色 token：1 主色 + 不透明度层级 */
  --c-bg: #2a3870;                        /* 背景色——每页都一样 */
  --c-fg: #E8D85C;                        /* 主文字色 */
  --c-fg-2: rgba(232, 216, 92, 0.62);    /* 二级文字（62% 不透明度） */
  --c-fg-3: rgba(232, 216, 92, 0.32);    /* 三级文字（32% 不透明度） */
  --c-accent: #3a7878;                    /* 单一强调色——仅用于极少量的点睛 */
  --c-border: rgba(232, 216, 92, 0.20);  /* 边框/分隔线（20% 不透明度） */
}
```

**关键模式：用不透明度变体代替多个颜色。** 不需要 `--text-muted`、`--text-subtle` 等独立变量——用同一个前景色的 0.62/0.32/0.20 不透明度版本。

### 颜色场背景

> 背景不是画布，它就是颜色本身。

```css
/* ✅ 颜色场：整个 slide 就是一种颜色 */
.reveal section { background: var(--c-bg); }

/* ❌ 不要在颜色场上加装饰纹理 */
/* 点阵、网格线、渐变叠加——这些是"在颜色上加东西"，不如选对颜色 */
```

浅色模板用纸质背景色（cream `#fafadf`、bone `#f0ece3`、ivory），不是纯白 `#fff`。深色模板用有色彩倾向的深色（深海军蓝 `#2a3870`、暖炭 `#1c2644`），不是纯黑 `#000`。

### 强调色极端克制

每个模板**只用 1 种强调色**，且使用频率极低：

- **仅用于**：引号标记、极少量关键词强调、CTA 元素
- **绝不用于**：背景渐变、大面积色块、装饰元素
- 主文字色和背景色的对比关系承担了 90% 的视觉层次

### 字重范围 200-600

```css
/* ✅ 优雅的字重范围 */
h1 { font-weight: 400; font-style: italic; }  /* 大标题用 italic，不用 bold */
h2 { font-weight: 600; }
p  { font-weight: 300; }                       /* 正文用轻字重 */

/* ❌ 避免 700-900 */
/* 粗重字重制造"厚重感"而非"存在感"。存在感来自字号和字间距，不来自字重。 */
```

### 强调通过反转实现

在 italic 风格的模板中，`<em>` 标签不是变斜体，而是反转：

```css
/* 正常文字是 italic，强调文字反转回 roman + accent 色 */
.display { font-style: italic; }
.display em { font-style: normal; font-weight: 600; color: var(--c-accent); }
```

这个技巧让强调色出现得极其自然——它只在需要被"读出来"的词语上。

### Pin 注释系统

每页左下角放置一个等宽字体的小注释，标注页码和主题信息。这是从印刷排版借鉴的"页码+章节标记"：

```css
.pin {
  position: absolute;
  bottom: 32px;
  left: 72px;
  font-family: var(--f-mono);
  font-size: 0.5em;
  color: var(--c-fg-3);   /* 最淡的层级 */
  letter-spacing: 0.06em;   /* 最淡的层级 */
  z-index: 10;
}
```

```html
<div class="pin">03 / 技术架构</div>
```

### 12 条设计原则总结

| # | 原则 | 做法 |
|---|------|------|
| 1 | 字体即人格 | 每个模板选择独特的显示字体组合 |
| 2 | 颜色即设计 | 整页颜色场，不加纹理叠加 |
| 3 | 单一强调色 | 只有 1 种 accent，使用频率 ≤5% |
| 4 | 不透明度层级 | 用同一颜色的 1.0/0.62/0.32/0.20 替代多个灰色 |
| 5 | 轻字重 | 200-600 范围，大标题用 italic 代替 bold |
| 6 | 纸质背景 | 浅色用 cream/bone/ivory，不用纯白 |
| 7 | 零装饰 | 没有装饰图形、玻璃拟态、侧边条带 |
| 8 | 命名页面类型 | Cover/Statement/Stats/Quote/Compare/End |
| 9 | Pin 注释 | 每页左下角等宽字体小字 |
| 10 | 强调反转 | italic 上下文中 `<em>` = roman + accent |
| 11 | Token 系统 | 所有视觉属性都是 CSS 变量 |
| 12 | 编辑式排版 | 想像印刷设计师，不想像前端框架 |

## 配色方案

### 完整 Token 配色系统

每个模板需要 6 个颜色 token。**不是 6 种颜色，是 1 种前景色 + 1 种背景色 + 不透明度变体。**

```css
:root {
  --c-bg: #2a3870;                        /* 背景色：每页统一，就是颜色场本身 */
  --c-fg: #E8D85C;                        /* 前景色：主文字、标题 */
  --c-fg-2: rgba(232, 216, 92, 0.62);    /* 二级层级：副标题、说明文字 */
  --c-fg-3: rgba(232, 216, 92, 0.32);    /* 三级层级：注释、pin 标注 */
  --c-accent: #3a7878;                    /* 强调色：唯一点缀，使用频率 ≤5% */
  --c-border: rgba(232, 216, 92, 0.20);  /* 边框/分隔线 */
}
```

**不透明度层级表：**

| 层级 | 不透明度 | 用途 |
|------|---------|------|
| 1.0 | 100% | 标题、正文、关键数据 |
| 0.62 | 62% | 副标题、说明文字、二级 UI |
| 0.32 | 32% | 注释、pin 标注、辅助信息 |
| 0.20 | 20% | 分隔线、边框、最淡的背景 |

### 背景色选择

背景是整个设计的灵魂——选对背景色，不需要任何纹理。

**深色方案：有色彩倾向的深色**

```css
/* ✅ 深海军蓝 */
--c-bg: #2a3870;

/* ✅ 暖炭灰 */
--c-bg: #1c2644;

/* ✅ 墨绿深林 */
--c-bg: #1a2f2a;

/* ✅ 深酒红 */
--c-bg: #2d1b2e;
```

**浅色方案：纸质色，不是白色**

```css
/* ✅ 奶油纸 */
--c-bg: #fafadf;

/* ✅ 骨色纸 */
--c-bg: #f0ece3;

/* ✅ 象牙白 */
--c-bg: #f5f0e8;

/* ✅ 暖灰纸 */
--c-bg: #edeae4;
```

**场景化配色建议：**

| 场景 | 背景色 | 前景色 | 强调色 | 感觉 |
|------|--------|--------|--------|------|
| 技术演讲 | `#1c2644` 深海军 | `#e2dcd0` 暖白 | `#c8a870` 古金 | 冷静权威 |
| 教学课件 | `#fafadf` 奶油纸 | `#1a1a16` 墨黑 | `#5a7a5a` 暗绿 | 温和亲切 |
| 产品发布 | `#2a3870` 深蓝 | `#E8D85C` 暖黄 | `#3a7878` 青绿 | 大胆自信 |
| 学术报告 | `#f0ece3` 骨色纸 | `#2a2520` 深棕 | `#8a6a4a` 焦赭 | 严谨典雅 |
| 创意展示 | `#1a1a2e` 深紫 | `#f0e6d0` 暖白 | `#d4a853` 琥珀 | 神秘高级 |

> ⚠️ **禁止使用 Tailwind 默认色板**（#6366f1、#8B5CF6、#3B82F6 等）。以上配色均使用自定义色值。

### 渐变背景

> 大多数页面不需要渐变。**纯色背景 + 正确的前景色组合**已经足够有表现力。

仅在章节分隔页或签名时刻页面使用渐变，且**仅使用单色相深浅渐变**：

```css
/* ✅ 单色相深浅渐变 */
background: linear-gradient(135deg, oklch(25% 0.03 250) 0%, oklch(15% 0.02 250) 100%);

/* ❌ 紫→蓝/蓝→青双色调渐变是 AI 标志性指纹，禁止使用 */
```

### 60-30-10 法则

- **60%**：中性背景、留白、基础表面
- **30%**：次要颜色——文本、边框、非活跃状态
- **10%**：强调色——CTA、高亮、焦点状态

### Accent 使用上限

强调色（`--accent`）每页最多出现 **3 次**（2 次更佳）。

典型组合：
- 一个 eyebrow/chip + 一个 CTA 按钮
- 一个卡片高亮 + 一个标签 pill

计数规则：
- 链接颜色算 accent；如果同一页有 CTA，链接降级为 `--text` + underline
- hover/focus ring 算 accent，需计入配额
- 渐变中的 accent 也算一次

### 调色板选择规则

- 每个演示文稿选定 1 套调色板，**全程不改、不混用**
- 严禁在生成过程中自行修改 hex 值
- 如果用户没指定，根据场景推荐并说明理由

## 字体系统

### 字体层级（保持对比）

| 角色 | 典型比例 | 用途 |
|------|---------|------|
| xs | 0.75em | 注释、法律文字 |
| sm | 0.875em | 次要 UI、元数据 |
| base | 1em | 正文 |
| lg | 1.25-1.5em | 副标题、引导文字 |
| xl+ | 2-4em | 标题、主文本 |

**原则：使用更少的尺寸，但对比更强。** 避免相邻尺寸太接近（14px, 15px, 16px...）。

### 字体组合选择

> **字体选择 = 模板人格。** 不用装饰元素区分模板，用字体组合区分。

每个模板选择 3 个字体角色：显示字体（标题）、正文字体、等宽字体（代码/注释）。

**有辨识度的显示字体组合：**

| 风格 | 显示字体 | 正文 | 等宽 | 感觉 |
|------|---------|------|------|------|
| 优雅社论 | Cormorant Garamond italic | DM Sans | Courier Prime | 印刷质感、editorial |
| 技术工业 | Jost 300 | Jost 400 | JetBrains Mono | 精确、工程感 |
| 温暖学术 | Source Serif 4 | DM Sans | IBM Plex Mono | 学术权威 |
| 简洁现代 | Outfit 300 | Outfit 400 | Fira Code | 清晰、专业 |
| 柔和自然 | Lora italic | DM Sans | Courier Prime | 有机、亲切 |

**必须避免的 AI 高频字体**（来自 impeccable 框架的 reflex-reject 列表）：

| ❌ 避免 | 原因 |
|--------|------|
| Inter | AI 训练数据频率 #1 |
| Roboto / Arial | 系统默认，零辨识度 |
| Plus Jakarta Sans | 2024-2025 AI 标配 |
| Fraunces / Newsreader | impeccable 明确禁止 |
| DM Serif / Syne | AI monoculture 代表 |

**中文字体固定搭配**：所有模板统一使用 Noto Serif SC（标题/引用）+ Noto Sans SC（正文），确保 CJK 排版质量。

```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@200;300;400;500;600&display=swap" rel="stylesheet">
<link href="https://fonts.googleapis.com/css2?family=Noto+Serif+SC:wght@300;400;600&display=swap" rel="stylesheet">
```

### Letter-spacing（最容易被忽略的规则）

这是 AI 生成设计中最常被跳过的规则。**无例外。**

| 场景 | letter-spacing |
|------|---------------|
| 正文 (14-18px) | `0`（默认） |
| 小字 (11-13px) | `0.01em` ~ `0.02em` |
| UI 标签、按钮 | `0.02em` |
| **ALL CAPS** | **`0.06em` ~ `0.1em`（必须）** |
| 标题 (32px+) | `0`（禁止负 tracking，参见硬规则 #4） |
| Display (48px+) | `0`（禁止负 tracking，参见硬规则 #4） |

**为什么这是硬规则：**

- 大标题用 `letter-spacing: 0` 保证投影/PPTX 混排稳定，渲染质量优先于美学微调
- ALL CAPS 不加 tracking → 字母间视觉拥挤，看起来业余
- 这两条是 AI 生成设计最可靠的"指纹"——做到就能拉开差距

```css
/* ALL CAPS 示例 */
.kicker {
  font-size: 11px;
  text-transform: uppercase;
  letter-spacing: 0.12em; /* 必须有 */
}
```css
/* ALL CAPS 示例 */
.kicker {
  font-size: 11px;
  text-transform: uppercase;
  letter-spacing: 0.12em; /* 必须有 */
/* 大标题示例（硬规则 #4：禁止负 tracking） */
h1 {
  font-size: 3em;
  letter-spacing: 0;
}
```

### 字重范围 200-600

> 粗重字重（700-900）制造"厚重感"而非"存在感"。存在感来自字号和字间距，不来自字重。

专业级演示文稿只用 4 个字重档位：

| 角色 | 字重 | 用途 |
|------|------|------|
| Display italic | 300-400 italic | 超大标题——用 italic 代替 bold，更优雅 |
| 正文 | 300-400 | 正文、说明文字 |
| UI/标签 | 500 | 按钮、标签、导航 |
| 强调 | 600 | 关键数据、CTA——最高的字重 |

```css
/* ✅ 优雅的字重系统 */
h1 { font-weight: 400; font-style: italic; }   /* italic 大标题 */
h2 { font-weight: 600; }                        /* 章节标题 */
p  { font-weight: 300; }                        /* 轻字重正文 */
strong { font-weight: 600; }                    /* 强调文字 */

/* ❌ 避免 */
h1 { font-weight: 700; }  /* 太粗重 */
h1 { font-weight: 900; }  /* 更糟 */
```

### 行宽限制

正文每行 **50-75 个字符**。CSS 安全默认值：

```css
p { max-width: 65ch; }
```

### 中文字体配置

```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@200;300;400;500;600&display=swap" rel="stylesheet">
<link href="https://fonts.googleapis.com/css2?family=Noto+Serif+SC:wght@300;400;600&display=swap" rel="stylesheet">
```

> 注意字重范围调整为 200-600，与整体字重系统一致。

### 中英文混排规则

演示文稿中经常出现中英文混排，以下是关键排版规则：

**1. 标点挤压**

```css
/* CJK 标点与英文之间的间距调整 */
body { text-autospace: ideograph-alpha; }  /* 未来标准 */
/* 当前方案：手动控制 CJK 标点后不加空格，英文与中文之间加一个空格 */
```

书写规范：
- 中文与英文/数字之间加**一个半角空格**：`使用 React 框架`（非 `使用React框架`）
- 中文标点后不加空格：`你好，世界`
- 英文标点后加空格：`Hello, world`

**2. 标点切换**

| 场景 | 使用标点 | 示例 |
|------|---------|------|
| 全中文语境 | 中文标点 | `这是中文标题，副标题。` |
| 全英文语境 | 英文标点 | `This is an English title.` |
| 中英混合、英文为主 | 英文标点 | `使用 React 18, 支持并发特性.` |
| 中英混合、中文为主 | 中文标点 | `前端框架 React，已成为行业标准。` |

**3. 行内代码与中文**

```
✅ 使用 `createApp()` 创建实例     ← 代码前后有空格
✅ 调用 `init()` 方法              ← 代码前后有空格
❌ 使用`createApp()`创建实例       ← 代码紧贴中文，不美观
```

**4. 括号规则**

- 全角括号用于中文语境：`新功能（测试版）`
- 半角括号用于英文/代码：`React 18 (Stable)`
- 括号内全英文用半角：`这是一个 API (Application Programming Interface)`

**5. 数字与中文**

- 数字与中文之间加空格：`共 12 个模块`、`增长 30%`
- 日期和单位除外：`2026年6月`、`3小时`

## 布局与空间

### 核心原则

1. **内容优先**：设计服务于内容，不是装饰大于内容
2. **克制美学**：避免过度设计，保持专业简洁
3. **一致性**：统一的配色、字体、间距系统贯穿全篇

### 视觉节奏

通过变化的间距创造视觉节奏——不是每个地方都用相同的 padding。

```css
/* ❌ 错误：相同间距 */
.card { padding: 1em; }
.section { padding: 1em; }

/* ✅ 正确：变化的间距 */
.card { padding: 1.5em; }
.section { padding: 4em 2em; gap: 2em; }
```

### 不要过度使用卡片

```css
/* ❌ 错误：一切都是卡片 */
/* 不要把所有内容都包装在卡片中 */

/* ✅ 正确：只在需要时使用卡片 */
.feature-card { background: var(--c-fg-3); border-radius: 8px; padding: 1.5em; }

/* ❌ 错误：卡片嵌套卡片 */
/* 这会制造视觉噪音，扁平化层次 */

/* ✅ 正确：扁平化结构 */
.section { background: var(--c-fg-3); padding: 2em; }
.subsection { border-left: 2px solid var(--c-accent); padding-left: 1em; }
```

### 使用 flex-wrap 实现自动换行

```css
.flow, .arch-row {
  display: flex;
  flex-wrap: wrap;      /* ✅ 自动换行比硬编码安全 */
  max-width: 100%;
  overflow: hidden;
}
```

## 文案清晰化（借鉴 /clarify）

> 演示文稿的文案问题不是"写得差"，而是"太空洞"。每页文字应该**传递具体信息**，不是制造"看起来很专业"的假象。

### 文案自检规则

| 问题 | 检测方法 | 修复 |
|------|---------|------|
| **模糊标语** | "提升效率""优化体验""赋能未来" | 替换为具体数字或场景："部署时间从 2h → 3min""新用户 3 步完成首次配置" |
| **行话堆砌** | 观众听不懂的术语出现在标题 | 标题用观众语言，术语放进正文或备注 |
| **信息不完整** | 列了"3 大优势"但没说**对观众意味着什么** | 每个优势后面加一句"这意味着…" |
| **被动语态** | "系统被设计为…" "数据将被…" | 改主动语态："系统…""你可以在…" |
| **空泛比较** | "更快""更好""更强大" | 给出参照物："比上一代快 3x""比竞品少 80% 配置步骤" |

### 每页文案硬限制

- 标题 **≤15 字**（中文）/ **≤8 词**（英文）
- 副标题/描述 **≤40 字** / **≤20 词**
- 列表项 **≤20 字** / **≤10 词**，每项只表达 1 个信息
- 全页文字总量 **≤120 字**（超出拆页）

### Speaker Notes 人情味

演讲者备注应该写**人会说的话**，不是论文摘要：

```
❌ "本页展示了 Q4 季度营收同比增长情况，主要驱动力为新产品线的市场渗透。"
✅ "看看这个数字——Q4 直接翻了倍。主要归功于 11 月上线的新产品线，团队当时连续加班了两周，这个结果值得庆祝。"
```

## 力度调节（借鉴 /bolder + /quieter）

> 优秀的设计不是"安全就行"，而是**该大胆时大胆，该克制时克制**。这一节给出具体的加减力操作指南。

### 什么时候该加力（/bolder）

以下信号说明设计太安全，需要加大胆：

| 信号 | 具体操作 | 示例 |
|------|---------|------|
| 所有页面"看起来都差不多" | 选 1-2 页做**大胆视觉决定** | 标题页字号翻倍（5em+）、用全出血图片、纯色背景 |
| accent 色几乎看不见 | 加大 accent 面积到 20-30% | 一整页用 accent 色做背景、标题直接用 accent 色 |
| 排版层级模糊 | 加大字号和字重对比 | 标题 4em/700 vs 正文 0.9em/400，差距拉到 3 倍以上 |
| 布局对称无聊 | 打破对称 | 左 30% 右 70% 分割、标题偏左 1/3 处、大区域留白 |
| 感觉"还行但记不住" | 加一个**只有这个演示才有的元素** | 定制数据可视化、品牌专属图标、一句话用超大字号占满全页 |

### 什么时候该减力（/quieter）

以下信号说明设计过了，需要收敛：

| 信号 | 具体操作 | 示例 |
|------|---------|------|
| 每页都有动画 | 动画减到 **≤30% 的页面** | 只在标题页、关键数据页、结尾页保留动画 |
| 多种颜色同时出现 | 颜色减少到 **≤3 种** | 主色 + 1 种中性色 + accent，其余全部去掉 |
| 卡片/装饰元素过多 | 去掉所有**不承载信息**的装饰 | 删除纯装饰线、渐变块、背景图案 |
| 文字太多 | 每页删减 **30-50%** | 只留核心观点，细节移到 speaker notes |
| 间距太紧 | padding/gap **增加 50%** | 从 1em → 1.5em，让内容呼吸 |

### 力度平衡原则

```
80% 克制执行 + 20% 大胆突破

克制 80% 的页面：
  · 统一排版系统
  · 一致的间距和配色
  · 可预测的布局节奏

大胆 20% 的页面（通常是 1-2 页）：
  · 一个"只有这个演示才有"的视觉决定
  · 足够大胆到让人截图分享
  · 但不脱离整体调性
```

## 视觉细节

### 不要使用玻璃拟态 everywhere

```css
/* ❌ 错误：装饰性玻璃拟态 */
/* blur效果、玻璃卡片、发光边框到处使用 */

/* ✅ 正确：目的性使用 */
.glass-panel {
  background: rgba(255, 255, 255, 0.1);
  backdrop-filter: blur(10px);
  border: 1px solid rgba(255, 255, 255, 0.2);
}
```

### 不要使用渐变文字

```css
/* ❌ 错误：用于"冲击力"的渐变文字 */
/* 这只是装饰，没有意义 */

/* ✅ 正确：纯色 + 排版层次 */
.metric-title {
  font-size: 3em;
  font-weight: 600;
  color: var(--text);
}
```

### 不要居中一切

```css
/* ❌ 错误：一切居中 */
/* 居中会看起来通用且无设计感 */

/* ✅ 正确：左对齐 + 非对称布局 */
.slide-content {
  text-align: left;
  padding-left: 2em;
}
```

## 反模式清单

> 以下是 AI 生成演示文稿时常见的"俗套设计"，应避免：

### P0 硬规则（必须 pass）

这些是可检查的具体规则，每条违反都是质量回退：

| # | 禁止 | 原因 |
|---|------|------|
| 1 | **Tailwind 默认 indigo** (`#6366f1`, `#4f46e5`, `#8b5cf6` 等) 作为 accent | AI 生成的标志性指纹，必须用 `--accent` |
| 2 | **双色调"信任"渐变**在 hero（紫→蓝/蓝→青/靛→粉） | 扁平面 + 排版层级远胜于装饰性渐变 |
| 3 | **Emoji 做功能图标**（✨🚀🎯⚡🔥💡 在标题/按钮/列表中） | 用 1.6-1.8px 描边的单色 SVG + `currentColor` |
| 4 | **Serif 绑定时标题用了 Sans** | h1/h2 必须用 `--font-display` |
| 5 | **圆角卡片 + 左侧彩色边框** | 经典"AI dashboard 瓦片"形状，去掉圆角或去掉左边框 |
| 6 | **捏造数据**（"10× faster""99.9% uptime"） | 必须来自用户输入或标注来源 |
| 7 | **占位文案**（lorem ipsum / 功能一/功能二/placeholder） | 空白区域用排版解决，不要编造文字 |

### 配色反模式

| ❌ 避免 | ✅ 替代方案 |
|--------|-----------|
| 纯黑 (#000) 或纯白 (#fff) | 添加 tint（哪怕 0.01 chroma） |
| 灰色文字在彩色背景上 | 使用背景色的深色版本 |
| 紫色到蓝色渐变 + 霓虹强调色 | 选择单一强调色 + 中性背景 |
| 默认深色模式 + 发光强调 | 选择适合内容氛围的配色 |

### 字体反模式

| ❌ 避免 | ✅ 替代方案 |
|--------|-----------|
| Inter、Roboto、Arial | Cormorant Garamond、Jost、Source Serif 4 |
| 系统默认字体用于标题 | 选择有辨识度的显示字体组合 |
| 相邻尺寸太接近 | 使用 Modular Scale，尺寸对比明显 |
| 700+ 字重 | 用 italic 代替 bold，或加大字号 |
| 同一个字体全家桶 | 显示字体 + 独立的正文字体，2 种字体增加层次 |

### 布局反模式

| ❌ 避免 | ✅ 替代方案 |
|--------|-----------|
| 一切都是卡片 | 只在需要区分层次时使用卡片 |
| 卡片嵌套卡片 | 扁平化结构，使用 border-left 等区分 |
| 相同尺寸的卡片网格重复 | 变化尺寸，或使用非对称布局 |
| 一切居中 | 左对齐 + 非对称布局 |
| 相同间距 everywhere | 创造视觉节奏，紧密vs宽松交替 |

### 视觉反模式

| ❌ 避免 | ✅ 替代方案 |
|--------|-----------|
| 到处使用玻璃拟态 | 目的性使用，如 modal 背景 |
| 圆角矩形 + 通用阴影 | 目的性阴影，或扁平设计 |
| 渐变文字 | 使用排版层次代替 |
| 装饰性小图表 | 只在传达有意义数据时使用图表 |

## 模板视觉 DNA

每个模板不只是"换个色"，而是有完整的视觉语言。以下为每个模板的设计决策速查。

> CSS 配方和装饰元素代码：`references/design-polish.md` → 模板专属装饰

### 速查表

| | editorial-serif | dark-tech | minimal-spatial | vibrant-gradient | nature-fresh |
|---|---|---|---|---|---|
| **核心意象** | 印刷排版、稿纸 | 深色颜色场、终端感 | 纸质背景、精确排版 | 大胆颜色、强对比 | 暖色纸、柔和曲线 |
| **背景** | 奶油纸 `#fafadf` | 深海军蓝 `#1c2644` | 骨色纸 `#f0ece3` | 深紫 `#1a1a2e` | 暖灰纸 `#edeae4` |
| **显示字体** | Cormorant Garamond italic | Jost 300 | Source Serif 4 | Outfit 300 | Lora italic |
| **正文字体** | DM Sans | Jost 400 | DM Sans | Outfit 400 | DM Sans |
| **字重范围** | 300-600 | 200-600 | 300-600 | 300-600 | 300-600 |
| **强调色** | `#8a6a4a` 焦赭 | `#c8a870` 古金 | `#2a3870` 深蓝 | `#d4a853` 琥珀 | `#5a7a5a` 暗绿 |
| **标题风格** | italic + 400 | 轻字重 + 大号 | 600 + `letter-spacing: 0` | 300 + 大号 | italic + 300 |
| **签名页** | 超大引言 italic | 单数字聚焦 | 全出血色块 | 非对称分割 | 超大引言 + 装饰引号 |

### 使用规则

1. **选定一个模板后，全程使用其装饰语言**——不混用不同模板的装饰元素
2. **渐变方向统一**——如全篇用 `135deg`，不中途改 `45deg`
3. **至少 3 页展示模板特有元素**（背景纹理 + 标题装饰 + 列表标记）
4. **签名页必须使用该模板的签名页配方**，不用其他模板的

## AI Slop 测试

**关键质量检查**：如果展示这个界面给某人并说"AI 做的"，他们会立刻相信吗？如果是，这就是问题所在。

一个独特的界面应该让人问"这是怎么做到的"，而不是"这是哪个 AI 做的"。

回顾上面的 **不要** 清单——这些都是 2024-2025 年 AI 生成作品的指纹。

## 加灵魂的公式

**80% 验证过的模式 + 20% 独特选择**。那 20% 应该花在：

- **一个大胆的视觉决定**——字体选择、一个颜色决定、一个意外的比例
- **有个性的文案**——"开始追踪" 胜过 "Get started"
- **一个让人记住的微交互**——按钮按下去 2px、数字跳动计数
- **一个只有用过产品的人才会放的细节**——快捷键提示、产品特色的状态标签

如果有人截图这个演示文稿，圈外人能认出这是哪个产品的——你有了灵魂。不能——你只是出了一个模板。
