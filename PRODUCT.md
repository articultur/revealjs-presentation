# Product

## Register

brand

## Users

This project serves presentation authors using Codex or Claude Code to generate browser-native slide decks that can also export to editable PPTX. The immediate audience is a demanding deck reviewer who judges the actual rendered presentation, not just source structure or lint output.

## Product Purpose

Produce single-file Reveal.js presentation decks that feel designed, not templated. Success means the HTML opens cleanly, presents with strong visual rhythm, survives browser screenshot QA, and exports a usable PPTX without requiring the viewer to install tools.

## Brand Personality

Kinetic, exacting, stage-ready. The work should feel like a launch-event art director and a rigorous QA engineer met in the same artifact.

## Anti-references

Avoid generic AI deck grammar: repeated card grids, soft ghost cards, tiny uppercase labels on every section as scaffolding, beige editorial sameness, Mermaid-looking diagrams, placeholder blocks, data without source notes, and layouts that only pass static checks while failing in real browser preview.

## Design Principles

1. Rendered truth beats source optimism: browser screenshots and exported artifacts decide quality.
2. One slide, one memorable claim: every page needs a dominant visual idea and a clear proof object.
3. Brand energy must be earned: color, motion, and scale should feel like Bilibili's culture rather than generic SaaS polish.
4. Verification is part of design: overflow, first-open visibility, fragment fullness, and PPTX export are design gates.
5. Skill outputs should teach taste: every workflow rule should prevent a real failure mode observed in generated decks.

## Form Factor

The near-term product should remain a skill-first system: it is strongest as a generation, taste, QA, and export kernel for browser-native decks. A separate Web/App experience becomes valuable when users need persistent projects, visual element editing, comments, tweak controls, hand-drawn revision requests, asset libraries, team review, and version history.

The durable direction is therefore a two-layer product:

1. Skill core: prompt-to-deck generation, HTML-native proof objects, design rules, screenshot QA, overflow checks, and PPTX/PDF export.
2. App shell: visual editing, element selection, parameter controls, comments, drawing-based revision requests, collaboration, assets, and project history.

## HTML-Native Advantages

Use browser-native capabilities when they prove the slide's claim: complex charts, 3D scenes, shader moments, physics interactions, 2D interaction, maps, and controlled presentation states. These capabilities must be verified in browser screenshots and have static fallback for exported PPTX.

## Accessibility & Inclusion

Respect reduced motion, preserve high contrast for body text, keep content readable at 1280x720 projection scale, and never hide core content behind first-click fragments.
